# Pass System

Standard pass labels for every game in the Math Game Studio OS.

---

## P0 — Prove the idea

**Question:** Is there a game worth building here at all?

**Use for:** role idea, math fit, basic loop sketch, "is this a real game or just an exercise"

**Exit when:** the role makes sense, the math can become the action, the loop is promising enough to prototype

---

## P1 — Prove the loop

**Question:** Can a player understand it, play it, and fail for understandable reasons?

**Use for:** tutorial basics, controls, basic pressure, basic consequence, loop clarity

**Exit when:** goal is clear, player can play start to finish, math is really the action, game is rough but real, next bottleneck is about quality not confusion

---

## P2A — Make the run matter

**Question:** Why would the player care, return, or learn from the run?

**Use for:** star ratings, debrief/review, streaks, visible mastery, stronger end-of-session payoff, session identity

**Exit when:** the run feels worth finishing, the end of a run teaches something or rewards something, the player has a reason to try again

---

## P2B — Make the challenge fair

**Question:** Does the game challenge the right thing at the right time?

**Use for:** speed tuning, overshoot consequences, pacing, progression walls, level role adjustments, pressure balancing

**Exit when:** difficulty feels fair, the game tests the math not the wrong skill, new demands are introduced in teachable steps, frustration comes from challenge not mismatch

---

## P3 — Make it feel alive

**Question:** Does it feel alive, memorable, and specific to its world?

**Use for:** humor, surprise, customer/incident personality, world reaction, status strips, celebration polish, emotional punctuation, stronger fantasy

**Exit when:** it no longer feels like a tool, you remember moments not just scores, success and mistakes feel emotionally distinct

---

## P4 — Make it worth replaying

**Question:** Why would someone come back after already understanding the game?

**Use for:** randomization, variation, advanced goals, optimization layers, replay hooks, unlock reasons, surprise events

**Exit when:** another run feels meaningfully different, mastery has depth, replay is not just repetition

---

## P5 — Make it ready to ship

**Question:** Is it ready to survive real use?

**Use for:** teacher controls, unlock codes, zero-friction launch, save behavior, cleanup, accessibility, final release checklist

**Exit when:** teacher can use it easily, student can launch it easily, it is stable enough to share

---

## How to diagnose

| Symptom | Pass |
|---|---|
| Player is confused | P1 |
| Player is not motivated or not learning from the session | P2A |
| Game is unfair, too easy, too fast, or badly tuned | P2B |
| Feels dead, pasted-on, generic, or emotionally flat | P3 |
| Gets old too fast | P4 |
| Annoying to launch, teach, or deploy | P5 |

## Naming rule

When starting work, say the game and the pass:

- "Bakery Rush P4"
- "Fire Dispatch P2B"
- "Unit Circle still needs P1"

Never say "let's do another pass" without a label.
