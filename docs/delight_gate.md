# Delight Gate

## Purpose
Prevent the OS from building games that are structurally sound but dead.

## Use this when
- Reviewing a new game concept after the Game Experience Spec
- Deciding whether to move from concept to prototype
- Deciding whether the concept has enough delight, fantasy, and stickiness to justify implementation

## Controls
- Concept approval
- Concept rejection
- Concept revision before build

## Do not use this for
- Late-stage polish
- Technical QA
- P2B tuning
- Release decisions

### Minimum threshold

A concept should not move into implementation if:
- more than 2 answers are "No"
- or more than 4 answers are "Partly"

### Mandatory fails

Do not build yet if either is true:
- no clear source of delight
- theme is pasted on instead of carrying the math
