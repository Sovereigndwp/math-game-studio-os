# Portfolio Strategy

## Summary
Three games reached P3 done. The OS proved it can produce playable, structured, improving games with clean pass discipline. But no current game was built with the stronger pre-build system (Game Experience Spec, Delight Gate, design checks). The OS is now stronger than the games it built. Time to apply that strength forward, not just sideways.

## Bakery Rush — paused (secondary candidate)

**Status:** P3 done. Charm is real. Customer personality works. Status strip works.

**Role:** Secondary candidate. Return only if a richer decision chain is designed (recipe scaling, ingredient substitution, order prioritization). The current single-step tap loop limits the ceiling.

**Do not:** Push to P4/P5 on the current foundation.

## Fire Dispatch — active (primary candidate)

**Status:** P3 done. P4A career persistence done. Subset-sum combination is the strongest math loop in the portfolio. Fantasy carries the math naturally.

**Role:** Primary candidate for demonstrating the OS can produce a high-standard game. Weaknesses are fixable: incident pool expansion, scoring hierarchy fix, adaptive difficulty, wind-down phase.

**Next:** P4B (incident pool expansion) → scoring hierarchy alignment → then evaluate for P5.

## Unit Circle — frozen (learning scaffold)

**Status:** P3 done (fragile). The OS learned valuable patterns (precision placement, tier gating, retry state machine, tolerance tuning).

**Role:** Learning scaffold. The pizza lab fantasy is too weak for flagship. A unit circle game could be great with a different fantasy — but that would be a new concept, not more passes on this one.

**Do not:** Invest further unless the core fantasy is reimagined from scratch.

## Criteria for the next new flagship concept

Before any new game moves to implementation, it must:
1. Pass the Game Experience Spec (core fantasy, feelings, stickiness, 3 memorable moments)
2. Pass the Delight Gate (≤2 "No" answers, no mandatory fails)
3. Pass the Fantasy Integrity Check (theme carries the math, not pasted on)
4. Pass the Satisfaction Loop Check (loop is satisfying to repeat)
5. Have a multi-step decision chain (not single-answer math)
6. Have a clear source of optimization above correctness
7. Have genuine replay variation (variable inputs that change the math)
8. Have at least one physical or world-grounded consequence for errors
9. Reference-extract from at least one strong exemplar before build
