# Snack Line Shuffle — Misconception Notes

These misconceptions must be linked in the repo misconception layer and represented in:
`artifacts/misconception_library/snack-line-shuffle-misconceptions.json`

## Canonical category mappings

- M4 leftmost digit dominance → `concept_confusion`
- M5 operation sign blindness → `representation_mismatch`
- M6 zero-subtraction overgeneralisation → `concept_confusion`
- M7 bigger number first = bigger result → `concept_confusion`

## M4 — Leftmost digit dominance
**Canonical category:** `concept_confusion`

The child ranks by the first visible digit rather than by the computed total.

## M5 — Operation sign blindness
**Canonical category:** `representation_mismatch`

The child treats plus and minus as decoration and attends only to the visible numerals.

## M6 — Zero subtraction overgeneralisation
**Canonical category:** `concept_confusion`

The child overgeneralizes from n+0=n and mishandles n-0 or similar zero relationships.

## M7 — Bigger number first = bigger result
**Canonical category:** `concept_confusion`

The child anchors on the larger numeral in the expression and assumes that expression must produce the larger total.

## Note
These are game-specific misconception instances and should be stored as per-game entries tagged to the canonical misconception taxonomy rather than added as new canonical taxonomy categories.
