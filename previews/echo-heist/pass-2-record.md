# Echo Heist — Pass 2 Record

## Pass purpose
Pressure and progression proof. Do classes feel different? Does difficulty climb? Does the player have real tactical choices?

## What was proved
- Classes feel mechanically distinct through activated abilities tied to math
- Difficulty climbs measurably across 3 districts
- Player has 3 tactical inputs beyond movement: ability (Space), gadget (Q), echo (R)
- 30 missions span the full curriculum arc the GDD specified
- Hints prevent frustration at the 11–13 age target without making math trivial

## What changed from Pass 1

### 1. Class abilities (Space key)
Each class now has an activated ability that requires solving a class-specific math problem in exchange for a temporary gameplay buff.

| Class | Ability | Math type | Buff |
|-------|---------|-----------|------|
| Hacker | Overclock | Percents / equations | Next 2 puzzles score 30% higher |
| Ghost | Soft Step | Fractions / angles | Instantly reduces noise meter by the solved amount |
| Runner | Burst | Rates / integers | 3s sprint at 2× speed with zero noise generation |

Each class has 5 unique ability prompts, randomly selected. Cooldowns: Hacker 45s, Ghost 30s, Runner 40s. Ability popup uses the class colour. Wrong answer wastes the cooldown.

### 2. Gadget: Noise Maker (Q key)
One per mission. Player places it at their current position. After a 2s arm delay, it emits loud noise (guards within 6 tiles investigate it instead of the player) for 4 seconds. Visual: pulsing dot, then expanding concentric rings during emission. Guards that investigate the gadget go back to patrol afterward — it's a one-use distraction window, not a permanent solution.

### 3. Audio feedback (Web Audio API)
Synthesised tones only — no external files.

| Event | Sound |
|-------|-------|
| Correct answer | Ascending two-tone chime (400→600 Hz) |
| Wrong answer | Low sawtooth buzz (150 Hz) |
| Vault lock cracked | Rising three-tone sweep (300→500→800 Hz) |
| Mission exit | Triumphant three-tone (400→500→700 Hz) |

M key mutes/unmutes globally. Audio initialises on first key press (browser autoplay policy).

### 4. Hint system (H key)
Press H during any math popup (room puzzle, vault lock, escape gate, ability) to reveal a hint. Two levels:
- **Hint 1**: math category + approach (e.g. "Percent decrease")
- **Hint 2**: setup / partial work (e.g. "18 × 0.75")

Cost scales by district: −50 / −75 / −100 score. Max 2 hints per prompt. Hints are tracked in results screen (total hints used visible to player).

Hint text is baked into every prompt object (`hint1`, `hint2` fields). Ability prompts, vault prompts, and escape prompts all have hints.

### 5. 30 missions across 3 districts
Missions 4–30 are generated by two systems:

**Prompt generator** (`generateDistrictMissions`): each district has a bank of 6 template types matching its curriculum focus. Templates parametrise numbers at runtime for variety. Each mission gets 6 room prompts, 2–3 vault prompts, 1 escape prompt.

| District | Missions | Curriculum focus |
|----------|----------|-----------------|
| Training Gallery | 1–10 | Integers, fractions, percents, rate-time |
| Camera Commons | 11–20 | Two-step equations, percent change, supplementary angles, ratios, fraction addition |
| Escape Lines | 21–30 | Multi-step equations, expected value, rate problems, percent increase, rounding |

**Level generator** (`generateLevel`): deterministic seeded procedural maps. Each call produces a unique but reproducible layout with:
- Border walls + 2–4 internal wall segments with gaps
- Scattered carpet/metal floor tiles
- Fixed spawn (bottom-left) and exit (top-right)
- Guard count scales 2/3/4 per district
- Camera count scales 1–2/2–3/3–4 per district
- 5–6 interactables (alternating door/terminal)

### 6. Difficulty scaling (DISTRICT_CONFIG)
A configuration object drives all difficulty knobs per district:

| Knob | D1 | D2 | D3 |
|------|----|----|----|
| Escape time | 60s | 50s | 45s |
| Guard speed mult | 1.0× | 1.2× | 1.4× |
| Heat gain mult | 1.0× | 1.3× | 1.6× |
| Hint cost | −50 | −75 | −100 |

`loadLevel` reads from `getDConfig()` so all values apply automatically when a mission is loaded.

## Updated controls
```
WASD       Move
Shift      Crouch (quieter + slower)
E          Interact (doors, terminals)
Space      Class ability
Q          Place gadget
R          Record/stop echo
H          Hint during math popup
M          Mute/unmute audio
Esc        Cancel math popup (costs heat)
```

## Known limitations at this pass
- Guard AI does not navigate around walls (guards walk through walls during investigate/chase). Acceptable for Pass 2; fix in Pass 3 or pre-shipping.
- Procedural levels can occasionally place interactables in wall tiles (max-tries fallback prevents hard freeze but placement may be suboptimal). Low frequency.
- District 2 and 3 missions use the prompt generator which produces varied but randomly seeded prompts on each page load. Not deterministic across sessions. Daily Contract mode (same seed for everyone) would need a session seed to be fixed.
- Audio does not play for vault key events (handleVaultKey does not call sfxCorrect/sfxWrong). Small omission.
- Mastery streaks not yet implemented.

## Deferred to Pass 3
- Visual effects for abilities (screen tint, particle trail on Burst)
- Mastery streaks (+5% buff after 3 correct answers in same skill)
- Focus mode (Tab: slow-time during math, costs score multiplier)
- Guard pathfinding (A* or simple wall avoidance)
- Sound for vault key events
- Mission-specific objectives enforced in scoring (e.g. "disable 2 cameras" objective tracked)
- Leaderboard / daily contract seed
- Meta-progression unlocks (Fraction Lens, Percent Wheel, Rate HUD tools)
